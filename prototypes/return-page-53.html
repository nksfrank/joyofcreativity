Card decline — handled inside the Stripe embed, never reaches this page

A card decline produces no server webhook (per Order lifecycle #35). Stripe's embedded Checkout shows its own inline error under the card field and keeps the customer on the checkout step to try another card — they never redirect here. We design nothing for this case: the order stays pending and is cleaned up by checkout.session.expired. Only an async failure (Klarna/Swish declined after leaving) fires async_payment_failed → the failed state above.
